# 단위 테스트(Unit Test)

	- 기능을 구현한 후 해당 기능이 원하는대로 작동하는지 보기 위해 작성하는 프로그램
	- 테스트 중 가장 작은 단위의 테스트에 속한다
	- 어떤 메서드를 실행했을 때 기대하던 값이 잘 나오는지 체크하는 방식으로 진행한다
	- 메서드 구현보다 테스트 코드를 먼저 작성하는 것을 TDD라고 한다
	
# 테스트 주도 개발(Test-driven development)

	- 조건을 만족하는 테스트 프로그램을 먼저 작성한 후 기능을 구현하는 방식
	- 테스트 단위로 기능을 구현하기 때문에 더욱 객체지향적인 개발이 가능하다
	- 익숙하지 않으면 시간이 오래걸려 효율이 떨어짐
	
# JUnit

	- 단위 테스트를 할 수 있는 여러 기능을 제공하는 자바의 라이브러리
	- 프로젝트 배포시 작성되어있는 모든 단위 테스트를 통과하지 못하면 배포가 중단되게끔 설정되어 있다
	

# JUnit

	@Test
	
	- 해당 메서드가 단위 테스트 메서드임을 표시하는 어노테이션
	
	@Ignore
	
	- 이 테스트를 무시한다
	
	@Before
	
	- 테스트 시작 전에 실행하는 메서드
	
	@After
	
	- 테스트 종류 후에 실행되는 메서드
	
# JUnist Assert 종류

	- 기대했던 결과가 나오면 테스트 통과, 결과가 다르면 테스트 실패
	- assertEquals() : 기대하던 값과 동일하면 테스트 통과
	- assertTrue() : 실행 결과가 true일 때 테스트 통과
	- assertFalse() : 실행 결과가 false일 때 테스트 통과
	- assertNull() : 실행 결과가 null일 때 테스트 통과
	- assertNotNull() : 실행 결과가 null이 아닐 때 테스트 통과
	- assertThrows() : 실행 결과 원하는 예외가 발생하면 테스트 통과
