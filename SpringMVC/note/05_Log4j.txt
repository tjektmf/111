# Log4j

	- 로그를 편리하게 남길 수 있는 여러 기능들을 제공하는 라이브러리
	- 각 로그마다 위험 등급을 설정하여 로그를 체계적으로 정리할 수 있다
	- 개발중에는 더 자세한 로그가 나오게 하고, 서비스시에는 운영에 관련된 로그만 나오게 하는 형식으로 활용할 수 있다
	
# Logger

	- 각 클래스에 장착하여 해당 클래스에서 발생하는 로그를 수집하는 객체
	- Logger가 로그들을 수집하여 Appender로 전달한다

# Appender

	- 수집된 로그들을 원하는 장소(콘솔, 파일, DB)에 출력하는 객체

# Layout

	- 로그를 출력하는 모양을 설정할 수 있는 객체
	
# Log4j의 Logging level (priority)

	- OFF : 모든 등급의 로그를 끔
	- FATAL : 아주 심각한 로그를 작성할 때 사용하는 등급 / FATAL 만 출력
	- ERROR : 에러 발생 로그를 작성할 때 사용하는 등급 / ERROR 보다 심각한 로그만 출력
	- WARN : 경고 로그를 작성할 때 사용하는 등급 / WARN 보다 심각한 로그만 출력
	- INFO : 일반적인 정보 로그를 작성할 때 사용하는 등급 / INFO 보다 심각한 로그만 출력
	- DEBUG : 디버깅 용도로 촘촘하게 로그를 작성할 때 사용하는 등급 / DEBUG 보다 심각한 로그만 출력
	- TRACE : 에러를 추적하기 위해 정밀하게 로그를 작성할 때 사용하는 등급 / 모든 로그가 출력됨
	- ALL : 모든 로그를 출력
